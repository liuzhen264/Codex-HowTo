# Governance

This project optimizes for usefulness, freshness, and trust.

## Maintainer Responsibilities

- keep version matrix current
- review official Codex changes
- preserve the learning-level system
- reject low-signal listicles
- require references for behavior claims
- label experimental features clearly

## Content Maturity

| Label | Meaning |
| --- | --- |
| stable | Behavior is documented and unlikely to change abruptly |
| evolving | Feature is documented but changing quickly |
| experimental | Use with caution; verify before relying on it |
| deprecated | Kept only for historical context |
