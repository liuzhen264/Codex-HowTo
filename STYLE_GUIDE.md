# Style Guide

## Voice

Clear, practical, and engineering-first.

Write like a senior teammate teaching a repeatable workflow.

## Rules

- Teach workflows, not isolated features.
- Include failure modes.
- Include version context.
- Avoid UI-specific steps unless verified recently.
- Prefer templates users can copy.
- Explain permission and data exposure risks.
