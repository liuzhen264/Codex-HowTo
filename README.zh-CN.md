# Codex HowTo

把 Codex 从代码助手，用成真正的工程协作者。

Codex 已经不只是一个会改代码的 CLI。它现在覆盖 CLI、Codex App、IDE、云端任务、`AGENTS.md`、Skills、Subagents、MCP、Hooks、Plugins、Automations、浏览器工作流、Computer Use、GitHub 工作流和团队治理。

这个项目不是普通教程合集，而是一套完整学习系统。

## 从这里开始

### 1. 找到你的等级

先完成 [FIND-YOUR-LEVEL.md](FIND-YOUR-LEVEL.md) 里的自测，或者在 Codex 环境里运行本仓库内置 skill：

```text
codex-self-assessment
```

你会得到基于当前能力的个性化路线图。

### 2. 按路线学习

学习路线见 [LEARNING-ROADMAP.md](LEARNING-ROADMAP.md)。

| 等级 | 身份 | 你将掌握 |
| --- | --- | --- |
| Level 0 | Curious Starter | 理解 Codex 能做什么，以及如何安全开始 |
| Level 1 | Solo Operator | CLI、App、IDE、AGENTS.md、审批、测试、日常开发 |
| Level 2 | Workflow Builder | Skills、可复用工作流、浏览器验证、review 闭环 |
| Level 3 | Agent Orchestrator | Subagents、MCP、Hooks、Plugins、Computer Use |
| Level 4 | Team Systems Designer | Automations、GitHub、团队治理、安全策略、插件分发 |

### 3. 边做边学

每章都包含：

- 学完能做什么
- 什么时候用
- 什么时候不要用
- 心智模型
- 步骤
- 可复制模板
- 实战练习
- 常见错误
- 成功标准
- 测验
- 官方引用
- 最后验证日期

### 4. 建立你的 Codex 系统

本项目包含：

- `AGENTS.md` 项目指令模板
- 可复用 Skills
- Subagent 定义
- MCP 配置模式
- Hooks 和 Automations 示例
- GitHub / PR 工作流
- 前端浏览器验证工作流
- 团队落地手册
- 测验与评分规则

### 5. 保持最新

Codex 变化很快。阅读任何章节前，先看 [VERSION_MATRIX.md](VERSION_MATRIX.md) 和 [CHANGELOG_WATCH.md](CHANGELOG_WATCH.md)。

当前基线：

- 最后调研：2026-05-02
- Codex CLI 基线：0.128.0
- 覆盖面：CLI、App、IDE、Cloud、GitHub、MCP、Skills、Subagents、Plugins、Automations

## 项目原则

Codex 最强的时候，不是你给它一句模糊命令，而是你给它：

1. 清晰项目指令
2. 有边界的权限
3. 可复用工作流
4. 可验证输出
5. 反馈闭环
6. 团队级约定

这个项目教的就是这六件事。
